METADATA SCHEMA:
- "document_id": The ID supplied by the caller, if any (integer, or null if not supplied)
- "vendor": The name of the store or business that issued the receipt — i.e. the seller or service provider. Do not use the name of any person or business mentioned only as the subject of a line item. (string, or null if not found)
- "item_description": A description of the product(s) or service(s) on the receipt. If the receipt contains 2 or fewer distinct line items, return each item verbatim from the receipt (lightly cleaned of OCR artefacts, joined with "; "). ONLY IF it contains more than 3 line items THEN return a brief summary instead (e.g. "Grocery shopping — 8 items"). If it is a subscription or utility, name the service. (string, or null if not found)
- "receipt_number": The primary identifier for this transaction. Accept any of these labels: "Receipt no.", "Receipt Number", "Receipt #", "Order Number", "Order no.", "Invoice Number", "Invoice no.", "Tax Invoice", "Transaction no.", "Transaction Number", "Reference no.", "Confirmation number", or similar. If multiple identifiers are present, prefer in this order: Receipt no. > Invoice/Tax Invoice no. > Order no. > Transaction no. > Reference no. Extract only the single value for that preferred field — do not combine values from multiple fields. (string, or null if not found)
- "purchase_date": The date and time of the transaction, normalised to ISO 8601. Include time if available (YYYY-MM-DDTHH:MM:SS), otherwise date only (YYYY-MM-DD). Drop timezone offsets and fractional seconds (e.g. "2025-11-23T14:30:00+10:00" → "2025-11-23T14:30:00"). Parse any human-readable format, e.g. "18 Feb 2026 09:30 PM", "4 Dec, 2025 at 10:25 AM", "30/11/2025 12:08:13 PM", "7 Sep 2025". (string, or null if not found)
- "currency": The ISO 4217 currency code, e.g. "AUD", "SGD", "USD". If no explicit code is present, infer from currency symbols, vendor location, or caller-supplied context. (string, or null if inference is not possible)
- "total_amount": The total dollar value of the transaction. For refunds, this will be negative. (number, or null if not found)
- "gst_amount": The GST (or equivalent tax) amount as a separate line item. For refunds, this will be negative. Set to 0 if GST is only mentioned as included in the total with no itemised amount. Set to null if it is unclear whether GST applies at all (e.g. an overseas receipt with no tax mention). (number, or null)
- "is_refund": Whether this receipt represents a refund or return. (boolean — true or false, or null if not determinable)
- "payment_method": The payment method used. Prefer the most specific value available (e.g. "visa" over "credit card"). Normalise to lowercase. Examples: "cash", "visa", "mastercard", "amex", "eftpos", "paypal", "bank transfer", "gift card". (string, or null if not found)
- "category": MUST be exactly one of: "electronics", "food & drink", "travel", "health & wellness", "home & garden", "school", "entertainment", "vehicle", "insurance", "other"
- "sub_category": A sub-classification within the category (e.g. "groceries", "networking gear", "software subscription", "fuel", "restaurant"). (string, or null if not determinable)
- "reasoning": A concise explanation of your thinking for non-obvious fields — particularly any that were set to null, any ambiguous values, OCR issues encountered, and how currency or vendor was inferred. Always populate this field.
